# ドル円・ゴールド・BTC ニュースまとめサイト ── ロードマップ

## コンセプト
通勤中に「今、何があったか」をサッと見られる、**ドル円・ゴールド・BTCに特化したニュースまとめサイト**。

- 投資助言ではなく、**値動きに影響しそうなニュースを「提示」するだけ**
- 売買を促さない／断定的な予想をしない（＝「投資助言・代理業」の規制を避ける設計）
- 各ニュースは「見出し＋出典リンク＋自分（AI）の言葉での短い要約＋どの銘柄にどう効きそうか」で構成

## 全体構成（アーキテクチャ）

```
GitHub Actions（3時間ごとに自動実行）
   ↓ ニュース取得（無料RSS）
   ↓ 銘柄ごとに整理（＋将来はAIで要約・時間軸分類）
   ↓ index.html を生成
   ↓ サイトのリポジトリに自動コミット
   ↓ GitHub Pages で公開
```

すべてGitHubのサーバー上で動くので、**自分のPCを落としても動き続ける**。

## コスト一覧

| 項目 | 費用 |
| --- | --- |
| GitHub Actions（実行環境） | 0円（無料枠：private リポジトリで月2,000分。本件は十分収まる） |
| GitHub Pages（サイト公開） | 0円 |
| ニュース取得（RSS） | 0円 |

**まずは Phase 1〜3 まで完全に0円**で作れる。お金がかかり始めるのは AI 要約（Phase 4）を入れてから。

## フェーズ別ロードマップ

### Phase 0 ── 準備
- **この案件専用の新しいリポジトリ**を1個作る（**Private** 推奨。既存サイトとは別に）
- ローカルに clone して、そのフォルダで `claude` を起動
- ※ GitHub アカウント・Node.js・Claude Code は導入済み前提

### Phase 1 ── 最小版（MVP）★まずはここ
- Python スクリプトが、無料の Google ニュース RSS から3銘柄のニュースを取得
- 「見出し＋出典名＋リンク＋日時」を銘柄ごとに並べた `index.html` を生成
- ローカルで実行 → 表示確認 → GitHub Pages で公開
- **AIなし・自動更新なし**（まず「取得→表示→公開」の土台を固める）
- コスト：0円

### Phase 2 ── 自動更新（3時間ごと）
- GitHub Actions の cron（`0 */3 * * *`）で1日8回、自動実行
- 取得 → `index.html` 更新 → 自動コミット → Pages 反映
- これで「PCを落としても勝手に更新」が完成
- コスト：0円

### Phase 3 ── 見た目（モバイル最適化）
- 通勤中に片手で読める、スマホ前提のシンプルなレイアウト
- 「最終更新時刻」「銘柄タブ／セクション」「読みやすい文字サイズ」
- フッターに免責（「本サイトは情報提供であり投資助言ではありません／投資判断は自己責任で」）
- コスト：0円

### Phase 4 ── AI要約・時間軸分類
- LLM API で、各ニュースを **原文をなぞらず自分の言葉で1〜2文に要約**
- 「短期／中期／長期」でどの銘柄にどう効きそうかをタグ付け
- API キーは GitHub Secrets に保存（コードに直書きしない）
- コスト：数十〜数百円/月（ここから変動費）

### Phase 5 ── 品質・運用
- 重複ニュースの除去、ノイズ記事のフィルタ、出典の多様化
- エラー時の通知（失敗したら LINE や GitHub Issue で気づける仕組み）
- コスト：ほぼ0円

### Phase 6 ── 集客・IP化
- SEO、アーカイブ、銘柄追加、ニュースレターやLINE一斉配信など
- ⚠️ **マネタイズ段階の注意**：有料化や「個別の売買助言」に踏み込むと金融商品取引法（投資助言・代理業の登録）に触れる可能性。あくまで「一般的な情報・ニュース提供」の枠を保つ。本格化前に金融庁の情報や専門家で確認すること。

## 横断的な注意点（最初から最後まで守る）

1. **著作権**：ニュース本文をコピーしない。見出しの大量転載もしない。「出典リンク＋自分（AI）の言葉での要約」に徹する。AIに要約させるときも「原文を言い換えること」を明示。
2. **投資助言にしない**：「買え／売れ」「上がる／下がる」と断定しない。「こういう材料が出た」という事実提示にとどめる。免責表示を必ず置く。
3. **セキュリティ**：API キーやトークンは必ず GitHub Secrets に保存。コードや公開ファイルに絶対に書かない。リポジトリは Private。
4. **ニュース源の規約**：使う RSS／API の利用規約（商用利用可か、帰属表示が必要か）を確認する。
5. **GitHub Actions の癖**：cron は数分ずれることがある（ニュース用途なら無問題）。リポジトリに60日間活動がないと定期実行が自動停止する点に注意。
