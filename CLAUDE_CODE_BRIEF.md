# Claude Code 指示書 ── ニュースまとめサイト

このファイルは、**Claude Code に読ませて／貼り付けて使う指示書**です。
※ Claude Code・Node.js・GitHub アカウントは導入済み前提。

---

## この指示書の使い方（おすすめの進め方）

「全部まとめてやって」ではなく、**1タスクずつ確認しながら**進めます。推奨手順は次の通り。

1. このファイル（`CLAUDE_CODE_BRIEF.md`）と `ROADMAP.md` を、**プロジェクトのフォルダに置く**。
2. そのフォルダで `claude` を起動。
3. 最初にこう伝える：
   `このフォルダの CLAUDE_CODE_BRIEF.md と ROADMAP.md を読んで、プロジェクトの全体像と制約を把握して。まだコードは書かないで、理解した内容を要約して。`
4. 把握できたら：`では「タスク1」だけをやって。終わったらローカルでの確認方法も教えて。`
5. 動作確認 → 問題なければ git commit → `次に「タスク2」をやって。` …と1つずつ進める。

> 直接プロンプトを貼ってもOKですが、**ファイルを置いて「読んで」と指示する方が楽**です（Claude Code が必要に応じて読み返せるため）。

## 1. 準備（自分でやること）
- **この案件専用の新しいリポジトリ**を1個作る（**Private** 推奨。既存サイトとは別に）
- ローカルに clone して、そのフォルダで `claude` を起動

---

## 2. 最初に読ませる／貼る「プロジェクト文脈」

ファイルを置いた場合は「読んで」で伝わりますが、貼る場合は以下を使ってください。

```
これから「ドル円・ゴールド・BTC ニュースまとめサイト」を一緒に作ります。全体像は次の通りです。

【コンセプト】
- 通勤中に「今、何があったか」をサッと見られる、ドル円・ゴールド・BTCに特化したニュースまとめサイト
- 投資助言ではなく、値動きに影響しそうなニュースを「提示」するだけ。売買は促さず、断定的な予想もしない。

【最終的な構成】
GitHub Actions（3時間ごと）→ 無料RSSでニュース取得 → 銘柄ごとに整理（将来はAIで要約） → index.html を生成 → 自動コミット → GitHub Pages で公開。
自分のPCを落としても動き続ける構成にしたい。

【守ってほしい制約】
- 著作権：ニュース本文をコピーしない。見出し＋出典リンク＋（将来は）自分の言葉での要約に徹する。
- セキュリティ：APIキーやトークンは必ず GitHub Secrets を使い、コードや公開ファイルに直書きしない。
- まずは最小限から作り、1ステップずつ確認しながら進めたい。動いたら次の機能を足す方針。
- 各ステップで、ローカルでの実行・確認方法も教えてほしい。

まずはタスク1（最小版）からお願いします。この後タスクを順番に渡します。
```

## 3. タスク1 ── 最小版（MVP）★まずここから

```
タスク1：最小版を作ってください。

【作るもの】
1. fetch_news.py
   - 無料の Google ニュース RSS から、3銘柄のニュースを取得する。
   - RSS URL の形式：https://news.google.com/rss/search?q={クエリ}&hl=ja&gl=JP&ceid=JP:ja
   - 銘柄とクエリは、コード上部の辞書で簡単に編集できるようにする。初期値の例：
       "ドル円":   "ドル円 OR USDJPY 為替"
       "ゴールド": "金価格 OR ゴールド OR XAU"
       "BTC":      "ビットコイン OR BTC 価格"
   - 各銘柄、最新の10件程度を取得。
   - RSS の解析には feedparser を使う。各記事から「見出し・リンク（出典URL）・出典名・公開日時」を取り出す。
   - ⚠️ 記事本文は取得・表示しない。見出しと出典リンクのみ扱う（著作権対応）。

2. index.html を生成
   - 3銘柄をそれぞれ見出し（セクション）に分け、その下にニュースを新しい順で並べる。
   - 各ニュースは「見出し（クリックで出典元へのリンク）」「出典名」「日時」を表示。
   - ページ上部に「最終更新：YYYY-MM-DD HH:MM（JST）」を表示。
   - スマホで読みやすいシンプルな見た目（最小限のCSSをHTML内に直接書く。外部依存なし）。
   - フッターに免責文：「本サイトは情報提供を目的としたものであり、投資助言ではありません。投資判断はご自身の責任で行ってください。」
   - 出力先は ./docs/index.html（GitHub Pages を /docs 公開にするため）。

3. requirements.txt（feedparser など必要なものを記載）

4. README.md
   - ローカルでの実行方法（依存インストール → python fetch_news.py → docs/index.html をブラウザで開く）
   - GitHub Pages の有効化手順（Settings → Pages → Branch: main / フォルダ: /docs）

【この段階でやらないこと】
- AIによる要約（後のタスク）
- 自動更新（後のタスク）
- 凝ったデザイン（後のタスク）

【完了条件】
- ローカルで `python fetch_news.py` を実行すると docs/index.html が生成され、ブラウザで開くと3銘柄のニュースが見出し＋リンクで表示される。
- APIキーやトークンは一切使っていない（このタスクでは不要なはず）。

作ったら、ローカルで動かして確認する手順を教えてください。
```

## 4. タスク2 ── 自動更新（3時間ごと）※タスク1が動いてから

```
タスク2：3時間ごとに自動更新する仕組みを GitHub Actions で作ってください。

【要件】
- .github/workflows/update.yml を作成。
- cron で3時間ごとに実行（0 */3 * * *、UTC）。手動実行（workflow_dispatch）もできるようにする。
- 処理：リポジトリを checkout → Python セットアップ → pip install → python fetch_news.py を実行 → docs/index.html に変更があれば自動コミット＆プッシュ。
- コミットには標準の GITHUB_TOKEN を使う（追加のトークンは作らない）。
- 変更がない時はコミットしない（空コミットを避ける）。

【完了条件】
- workflow を手動実行すると、docs/index.html が更新されてコミットされる。
- 以後、3時間ごとに自動で更新される。

注意：GitHub Actions の cron は時刻が数分ずれることがありますが、本件は問題ありません。
```

## 5. タスク3 ── モバイル最適化 ※後で

```
タスク3：通勤中にスマホで片手で読める見た目に整えてください。

【要件】
- スマホ幅前提のレイアウト。文字サイズ・行間・余白を読みやすく。
- 3銘柄を切り替えやすく（タブ or 明確なセクション見出し）。
- 各ニュースカードに、見出し・出典・相対時刻（例：2時間前）。
- ダークモードに対応（任意）。
- 外部ライブラリは使わず、HTML内のCSSで完結させる（依存を増やさない）。
- 免責文と最終更新時刻は引き続き表示。
```

## 6. タスク4 ── AI要約・時間軸分類 ※コストが発生する段階

```
タスク4：各ニュースに、AIによる短い要約と時間軸タグを付けてください。

【要件】
- LLM API（Anthropic の軽量モデルなど）を使って、各ニュースの見出しから「原文をなぞらず、自分の言葉で1〜2文の要約」を生成する。
- あわせて「短期／中期／長期」のどれに影響しそうかをタグ付けし、どの銘柄（ドル円／ゴールド／BTC）に効きそうかを一言添える。
- 出力は「事実の提示」にとどめ、「買え／売れ」などの助言や断定的な予想はしない。
- APIキーは GitHub Secrets（例：ANTHROPIC_API_KEY）から読み込む。コードに直書きしない。workflow にも secrets 経由で渡す。
- コスト対策：1回の実行でAPI呼び出し回数・トークンを抑える工夫（まとめて処理する等）を入れる。

【完了条件】
- 各ニュースに、自分の言葉の短い要約・時間軸タグ・対象銘柄が付く。
- APIキーはコードや公開ファイルに一切現れない。
```

## 7. Claude Code と作業するコツ
- 一度に詰め込まず、**1タスクずつ**渡して、動作確認してから次へ。
- 各タスクの後、ローカルで実際に動かして確認する（`python fetch_news.py` → `docs/index.html` を開く）。
- うまく動いたら git commit。区切りごとに残しておくと戻れる。
- 期待と違う動きをしたら、エラーメッセージや画面の様子をそのまま Claude Code に伝えて直してもらう。
- 「なぜこう書いたのか」「他の選択肢は？」と聞けば解説してくれる。未経験のうちは積極的に質問するとよい。

## （任意）さらに確実にしたい場合：CLAUDE.md を置く
Claude Code は、リポジトリ直下の `CLAUDE.md` を自動で読み込み、毎回の作業方針として参照します。
以下の内容で `CLAUDE.md` を作っておくと、制約を毎回守らせやすくなります。

```
# プロジェクト方針
このリポジトリは「ドル円・ゴールド・BTC ニュースまとめサイト」です。
詳細は ROADMAP.md と CLAUDE_CODE_BRIEF.md を参照。

## 常に守る制約
- 著作権：ニュース本文をコピーしない。見出し＋出典リンク＋自分の言葉での要約に徹する。
- セキュリティ：APIキー・トークンは GitHub Secrets を使い、コードや公開ファイルに直書きしない。
- 投資助言にしない：「買え／売れ」「上がる／下がる」と断定しない。事実の提示にとどめる。
- 作業は1タスクずつ。指示されたタスクのみ行い、勝手に先のフェーズを進めない。
```
